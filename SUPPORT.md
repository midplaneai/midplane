# Support

Thanks for using Midplane. Here's where to go for help:

- **Midplane Cloud — account, billing, or anything else** → [support@midplane.ai](mailto:support@midplane.ai). A human reads and answers. Also reachable in-app: the "Email support" link in the sidebar, and the contact links on error pages.
- **Questions, ideas, usage help** → [GitHub Discussions](https://github.com/midplaneai/midplane/discussions)
- **Bugs & feature requests** → [open an issue](https://github.com/midplaneai/midplane/issues/new/choose) (use the templates)
- **Security vulnerabilities** → **don't** open a public issue; see [SECURITY.md](./SECURITY.md)
- **Commercial / Enterprise (`ee/`) licensing** → sales@midplane.ai, or [book a call](https://calendar.app.google/NaSZgsxq9ptBYrLy6)

Community support is best-effort. Self-hosting is a single supported artifact
(`MIDPLANE_SELF_HOST=1`); we can't debug arbitrary custom deployments — a
self-host build's in-app help link points to GitHub issues for the same reason.
The managed cloud is the fully supported path.
